     185             :  * chooses the conformal factor

     188             :  *   \psi = 1
     189             :  *   \text{,}
     190             :  * \f}
     191             :  *
     192             :  * so the spatial metric of the `GrSolution` is used as conformal metric,
     193             :  * \f$\bar{\gamma}_{ij = \gamma_{ij}\f$. This is particularly useful for
     194             :  * superpositions, because it means that the superposed conformal metric of two
     195             :  * `WrappedGr` solutions is probably a good conformal background to solve for a
     196             :  * binary solution (see Xcts::AnalyticData::Binary).
     197             :  *
     198             :  * For example, when the `GrSolution` is `gr::Solutions::KerrSchild`, the
     199             :  * conformal metric is the spatial Kerr metric in Kerr-Schild coordinates and
     200             :  * \f$\psi = 1\f$. It is also possible to
     201             :  * choose a different \f$\psi\f$ so the solution is non-trivial in this
     202             :  * variable, though that is probably only useful for testing and currently not
     203             :  * implemented. It should be noted, however, that the combination of
     204             :  * \f$\psi=1\f$ and apparent-horizon boundary conditions poses a hard problem to
     205             :  * the nonlinear solver when starting at a flat initial guess. This is because
     206             :  * the strongly-nonlinear boundary-conditions couple the variables in such a way
     207             :  * that the solution is initially corrected away from \f$\psi=1\f$ and is then
     208             :  * unable to recover. A conformal-factor profile such as \f$\psi=1 +
     209             :  * \frac{M}{2r}\f$ (resembling isotropic coordinates) resolves this issue. In
     210             :  * production solves this is not an issue because we choose a much better
     211             :  * initial guess than flatness, such as a superposition of Kerr solutions for
     212             :  * black-hole binary initial data.
